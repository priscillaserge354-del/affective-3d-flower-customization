# 情绪花园项目：深度研发攻坚与调试备忘录 (2025.06 - 2025.12)
**记录单位**：WeChat Garden 项目开发组
**文档版本**：v2.0.4 (Final Stable)
**说明**：本日志详尽记录了项目在 2025 年下半年关键开发迭代期中，团队针对 3D 渲染瓶颈、多模态算法融合、跨平台资产兼容性等核心技术瓶颈的攻坚过程。

---

### 阶段一：3D 场景底层架构重构与渲染性能调优 (2025.06 - 2025.07)

#### [Issue #A-102] 大规模 3D 对象的显存溢出与 Draw Call 优化
* **技术背景**：在最初的 Alpha 版本中，团队尝试为日历方阵中的 365 天分别创建独立的 Mesh 对象。由于每个模型包含复杂的顶点数据，当渲染进度超过 100 天时，显卡 Draw Call 次数激增，导致中低端设备直接崩溃。
* **攻坚过程**：
    1.  **性能监测**：通过 Chrome 开发者工具的 WebGL 性能面板发现，瓶颈在于频繁的 CPU-GPU 数据交换。
    2.  **方案论证**：团队紧急召开技术研讨会，对比了“静态合并（BufferGeometryUtils）”与“动态实例化（Instanced Mesh）”方案。
* **最终方案**：全面引入 **Instanced Mesh (实例化网格)** 技术。通过共用一套底层几何体与材质数据，仅在 GPU 层面通过变换矩阵（Matrix4）修改位置与缩放。
* **成果评估**：成功将系统资源占用降低了 85% 以上，支撑了 3000+ 草丛个体在 60FPS 下的丝滑漫游，实现了全年度花园的宏大视觉表达。

### 阶段二：多模态交互逻辑闭环与异步状态管理 (2025.08 - 2025.09)

#### [Issue #B-205] 语音识别状态机的逻辑竞争与 UI 堆叠异常
* **现象描述**：在 Beta 1.0 版本的集成测试中，点击录音图标后，若网络存在波动，前端会偶发性地直接跳过“聆听”阶段，直接显示空的分析结果，导致多个 UI 层级互相穿透、堆叠。
* **深度分析**：原有的事件触发机制属于“同步阻塞式”，没有考虑到 AI 模型接口响应的异步延迟，导致状态转换逻辑混乱。
* **解决方案**：
    1.  **状态机重写**：构建了严格的“有限状态机 (FSM)”，将交互流程划分为：`Idle(空闲)` -> `Listening(录音)` -> `Analyzing(计算)` -> `Result(展现)`，并设置互斥锁。
    2.  **异步计时器机制**：独立编写了 `setInterval` 封装函数，确保秒表计时与声纹波形分析动画同步进行，并配合 `Promise` 机制等待后台算法回传。
* **成果评估**：彻底解决了录音模块的逻辑冲突，实现了“发送并开始声纹分析”的沉浸式动态反馈。

### 阶段三：算法精度打磨与大学生专用语义词库构建 (2025.10 - 2025.11)

#### [Issue #C-301] 语义识别的“语境缺失”与智能化表现修缮
* **问题背景**：基础 AI 模型在处理校园俚语（如“早八”、“查分”、“挂科”）时，往往只能给出中性判定，无法准确映射至对应的花卉色彩，被测试组吐槽为“人工智障”。
* **团队行动**：开发组决定不完全依赖云端黑盒 API。团队全员参与，手动整理并构建了《大学生高频情绪关键词权重表 v2.0》，收录词条达 500 余条。
* **技术突破**：在 Javascript 逻辑层插入了一个“微型预处理器”。
    * **加权匹配算法**：引入关键词分值系统，对“外卖被拿”、“深夜 DDL”等特定场景赋予极高的情绪权重。
    * **多路径判定**：采用 `some` 和 `includes` 的多重逻辑过滤，实现了即便在离线或弱网状态下，系统也能通过本地词典快速、精准地自动选中对应情绪。
* **成果评估**：极大增强了系统的“人情味”，使 3D 花园的情绪反馈更符合大学生的真实生活心境。

### 阶段四：资产流水线自动化与后端诊断系统开发 (2025.11 - 2025.12)

#### [Issue #D-408] 资产路径转义失效与 3D 模型 404 顽固 Bug
* **现象**：由于 GLB 模型文件夹命名规范（如 `F M 5` 中的空格）在不同操作系统和 Flask 静态路径下解析不一致，导致新增的情绪花卉频繁显示为默认占位球体。
* **修复措施**：
    1.  **诊断工具升级**：修正了后端 Python (`app.py`) 诊断脚本。将原有的、仅针对 `.gltf` 的模糊扫描升级为针对二进制 `.glb` 文件的指纹特征识别，并在服务器启动时自动打印资源完整性报告。
    2.  **容错机制（Fallback Strategy）**：开发了“降级渲染逻辑”。当外部 3D 资产因网络或路径原因加载失败时，系统会自动捕捉错误（catch error），并实时生成带对应情绪色的几何多面体进行占位，确保花园视觉绝不留白。

### 阶段五：演示前夕的交互细节压测与 UI 磨合 (2025.12)

#### [Issue #E-502] UI 穿透引发的 Canvas 事件冲突
* **现象**：用户在点击“关闭日历”或“调整滑块”时，点击事件会直接穿透透明背景，误触底下的 3D 花朵模型，频繁触发不必要的弹窗。
* **最终方案**：
    1.  **事件拦截**：对所有的“玻璃拟态”前端面板应用了严格的 `e.stopPropagation()` 策略。
    2.  **视口对齐逻辑**：针对宽屏显示器，重新计算了 3D 场景的 `camera.aspect` 与 `renderer.setSize` 映射关系，确保花园日历始终处于构图的视觉重心。
* **成果评估**：完成了最后的 UI/UX 细节打磨，项目进入稳定交付状态。
